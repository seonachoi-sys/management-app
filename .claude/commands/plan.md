# Plan 작성 워크플로우

새 기능 구현 또는 대규모 수정 전에 이 워크플로우를 따라 계획을 수립합니다.

## 1단계: 현황 파악
- 관련 파일 전체 읽기 (컴포넌트, 서비스, 타입, 훅)
- 기존 데이터 구조 확인 (Firestore 컬렉션 + 타입 정의)
- 영향받는 컴포넌트 목록 작성

## 2단계: 요구사항 정리
- 사용자 요청을 구체적 기능 단위로 분해
- 업무관리 / 과제관리 어느 쪽에 해당하는지 구분
- `/settings` 연동 필요 여부 확인
- 기존 Firestore 컬렉션으로 충분한지, 새 컬렉션 필요한지 판단

## 3단계: 구현 계획 작성

### 계획서 구조
```
## 목표
[한 줄 요약]

## 수정 파일
| 파일 | 변경 내용 | 신규/수정 |
|------|----------|----------|

## 데이터 변경
- Firestore 컬렉션: 추가/수정 여부
- 타입 정의: 새 필드/인터페이스

## 구현 순서
1. [타입 정의]
2. [서비스 레이어]
3. [커스텀 훅]
4. [UI 컴포넌트]
5. [CSS]

## 주의사항
- [기존 코드 영향]
- [날짜 처리 규칙 준수]
- [Firestore 규칙 준수]

## 테스트 체크리스트
- [ ] 타입 체크 통과 (npx tsc --noEmit)
- [ ] 기존 기능 깨지지 않음
- [ ] 모바일 반응형
- [ ] 한국어 UI
```

## 4단계: 승인 후 구현
- 계획 공유 → 사용자 확인 → 구현 시작
- 구현 중 계획 변경 필요 시 사용자에게 알림
- 완료 후 체크리스트 검증

## 과제 관리 특수 규칙
- 참여율 변경 시: 3책5공 검증 + 100% 초과 검증 필수
- 인건비 계산 변경 시: 현금/현물 비율 + 퇴직금추계 로직 확인
- 예산 변경 시: budgetDetail 구조 + RCMS 매핑 호환성 확인
- 직원 데이터 변경 시: 인건비 산출에 영향 → 재계산 필요 여부 확인
